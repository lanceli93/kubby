我想要参考jellyfin自己构建一个本地影音系统, 计划是先实现最小功能, 这个应用的名称叫kubby, 计划也是支持多种媒体库, 但目前只需要支持电影, 技术栈我是想用next.js + shadcn/ui + tailwind 做前端, 后端还没确定, 由于是本地影音系统, 我计划先用sqlite做数据库, 后端也先做简单实现, 用与next.js配套的技术栈, 后面如果要云端拓展再重构.

目前能想到的基本功能:
1. 注册用户, 登录用户
2. 电影媒体库创建 (提供文件夹)
3. 电影元数据提取, 其中nfo文件已在文件夹下. 一个典型的目录如下
- folder
  - film1
    - film1.mp4
    - movie.nfo
    - poster.jpg
    - fanart.jpg
  - film2
    - film1.wmv
    - movie.nfo
    - poster.wepb
    - fanart.png
4. 其他媒体库类型在框架中留位置, 但暂时不实现
5. 前端主要web 页面需要实现, 如首页, 以及控制台, 账户设置等
6. 后端实现基本功能, 播放器直接播放本地视频, 以后再拓展流播放, 电视直播, 插件支持等复杂功能暂时不做
7. 目前能想到的重要功能: 媒体详情页, 左边是poster, 上方有fanart, 有显示元数据,以及演员照片. 点击演员照片可以跳转到此演员参演的电影.

我计划先使用pencil mcp(一个用于Web UI设计的工具)生成所需 web 页面等template, 采用深色影院+卡片风格. 然后在做功能开发等. 

使用jellyfin 这个skill作为参考, 帮我丰富上面提到的需求,并确认技术栈, 使之更完整且合理. 初始版本尽量避开复杂功能, 只完成最基础最重要的功能, 但需要留空间给后续进行功能扩展.
